CopyOnWriteArrayList

发表于 2022-03-26 更新于 2022-03-29 分类于 Java

UML类图

写入时复制策略

当有多个调用者同时请求一个资源数据时，某个调用者需要对数据进行修改，此时复制一个当前资源数据的副本给调用者修改

  • 并发场景下，进行写操作加锁， 进行读操作不加锁，读写分离

普通集合的并发修改异常, 如何在遍历时修改集合元素?

  1. modCount变量: 对集合进行写操作时, modCount++
  2. 使用Iterator迭代器遍历集合时, expectedModCount会保存当前modCount的值, 如果hasNext()方法为true, next()方法会校验expectedModCount与modCount的值, 如果不相等, 抛出ConcurrentModficationException

CopyOnWriteArrayList的特点

  • 采用写入时复制策略, 读写分离, 读和写操作分别作用于不同的List容器
  • 写操作加锁, 读操作没有加锁, 适用于读多写少的应用场景
  • 由于读写分离, 遍历CopyOnWriteArrayList时不会抛出ConcurrentModificationException 遍历CopyOnWriteArrayList
  • 数据量大时, 由于写操作会拷贝数据, 内存会有较大的压力, 可能会引起频繁Full GC
  • 如果在写操作时进行读取, 此时读取的数据是旧容器的数据
